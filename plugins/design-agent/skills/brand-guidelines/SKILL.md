# brand-guidelines

Compile the rules that let anyone use the brand correctly without you in the room. **Act as a Senior Brand & Graphic Designer.** A brand book's job: make the right use easy and the wrong use obvious.

Design WORK — runs late, after `/brand-identity`, before `/create-report`. Inherits the **principal-grade standard**. Pull precise values from Figma via the Figma MCP if connected. Work in chat; nothing posts until approved.

## Inputs

The full identity system (`/brand-identity`), logo (`/logo-design`), art direction (`/moodboard`), and strategy (`/brand-strategy`).

## Method (in order)

1. **Brand essence (brief).** Positioning, personality, and the one-line "what we stand for" — so users design *with intent*, not just to-spec.
2. **Logo.** Versions/lockups, clear space, minimum size, correct color treatments, placement — and a **do/don't** with real misuse examples (stretched, recolored, on busy backgrounds).
3. **Color.** Palette with roles + exact values (HEX/RGB, and CMYK/Pantone for print) + pairing/contrast rules.
4. **Typography.** Families, scale, weights, hierarchy, usage rules (web + fallback).
5. **Imagery & illustration.** Style rules + do/don't that keep visuals on-brand.
6. **Layout & motifs.** Grid, spacing, signature graphic elements, iconography.
7. **Voice (brief).** Tone principles (or pointer to `/ux-writing`).
8. **Applications.** A few real examples (card, social, header) showing the system working together.

## Senior rigor (always)

- **Rules, not just specimens:** each section says how to use it AND how not to.
- **Show misuse:** do/don't with concrete wrong examples prevents the predictable mistakes.
- **Precise:** exact values (print + digital), no "about".
- **Usable cold:** a stranger could brand-correctly produce a piece from this alone.
- **Open questions:** anything still undecided.

## Output

**Start from the scaffold in `references/template.html`** — copy it and fill in every placeholder (keep the house style). It produces a self-contained **HTML document** (inline CSS, rendered specimens + do/don't) structured as a brand book: Essence · Logo (+ do/don't) · Color · Typography · Imagery · Layout & motifs · Voice · Applications. Show it in chat, iterate, then hand to `/create-report`.
